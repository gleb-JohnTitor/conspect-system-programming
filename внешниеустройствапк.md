# Внешние устройства

Центральный процессор + память, с точки зрения информатики, = компьютер, но от него никакого толка бы не было.

Потому что:

1. Компьютеру надо как то получать программы пользователей и отдавать их обратно, в принципе это весь смысл компьютера
2. Ячейки памяти (RAM) теряют свои значение после отключение компьютера от электричество

В ранних компьютерах было все непосредственно напрямую к CPU, но отказались ведь это было не удобно т.к процессор приходилось переделывать под новые устройства, тогда и возникла идея **общей шины**. Подключение всех внешних устройств к общей шине производится через ***контроллер*** - электронная схема, которая может взаимодействовать через шину с CPU, все контроллеры это делают одинаково. С точки зрения CPU - они все одинаковые. Взаимодействие происходит через команду *запись* и *чтение*. CPU не знает имеет дело с контроллером или с чем то другим поэтому у них есть **адреса портов ввода-вывода**

Работа процессора аналогична что и с ячейками памяти, но различие все же есть:

*Таблица (1. сравнение операций над RAM и Контроллеров)* 

| Операция | RAM | Контроллеры |
| --- | --- | --- |
| Чтение | При чтение ячейки памяти выстраивают логическое уравнение, чтобы CPU смог прочитать данные из ячеек RAM ----> CPU | При чтение не выдают какое то сохраненное значение, а передает какое-то значение которое связанно с состоянием самого контроллера, например, завершена ли какая то операция |
| Записи | При записи  дорожки должны выстроить логическое уравнение для дорожкам CPU ----> RAM | При записи значение они не воспринимают, а воспринимают как команду что-то сделать |

 Сколько портов, какие предписание можно ему давать и т.д  - ничего не знает об этом CPU, все как правило зависит от самого контроллера. Для работы с конкретным контроллером шинам нужна специальная программа ***драйвер*** - дает самостоятельно команды записи и чтение портов ввода-вывода для своего контроллера. Драйвер устанавливает самостоятельно при установке/запуске ОС. 

Например, если программе потребовалась сохранить файл на диске, то программа обратиться к ОС, она в свою очередь вычислит где будет находится файл на диске. ОС обращается к драйверу с просьбой записать файл на диск. Драйвер обратится к своему контроллеру с командой, сначала выполнит операцию записи на порты ввода-вывода, что заставит контроллер начать операцию записи; затем, выполнив операцию чтения из портов, узнает о результатах операции и сообщит о них ОС.
